## 真实浏览器流程

- 注册新用户后返回登录页，并显示注册成功提示。
- 使用新账号登录后进入工作台，昵称、角色和后端健康状态显示正确。
- 危险写操作问题被后端拒绝，页面展示中文业务原因。
- 自然语言查询成功返回审核后的 SQL、动态数据表格和 AI 业务总结。
- 查询历史同时记录成功查询和安全拒绝记录，并且只展示当前用户的数据。
- 历史详情可以查看问题、SQL、审核结果、耗时和 AI 总结。
- “再次提问”会回填原问题，但不会自动执行，不会意外消耗模型额度。
- 点击用户信息可以稳定展开退出菜单，退出后受保护路由会跳转到登录页并保留原目标地址。

## 本次修复

### 1. 业务错误提示统一

Axios 遇到 HTTP 400 时原本会把英文 `Request failed with status code 400` 传给页面，虽然全局消息已经显示后端中文原因，但页面错误卡片仍不友好。

修复后保留原始 Axios 响应对象，同时把异常消息统一为后端返回的业务消息，页面错误态与全局提示保持一致。

### 2. 退出菜单改为点击触发

用户菜单原本使用 Element Plus 默认悬停触发，在触屏设备和自动化场景下不稳定。现在显式改为点击触发，退出入口更明确，也更容易测试。
